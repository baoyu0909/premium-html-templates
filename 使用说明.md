# Premium HTML Templates Skill 使用说明

这个技能包用于生成高贴合度的 HTML 汇报页、PPT 风格 deck、单页 dashboard 和信息卡页面。它沉淀了三类可复用视觉语言，并额外区分了「PPT 分页模式」和「单页模块模式」，方便不同场景稳定产出。

## 适合做什么

- 生成 HTML 版 PPT、汇报、复盘、提案、规划方案
- 生成单页数据看板、信息卡页面、资源盘点页
- 生成接近既定风格的配色、版式、卡片结构和标题层级
- 后续扩展新的风格类型，而不破坏已有规则

```text

│   ├── archetypes-index.md

│   └── variant-light-report.md
└── .agents/
    └── skills/
        └── cose-html-theme-replica/
```

根目录的 `SKILL.md` 和 `references/` 是导出、分享、导入时最重要的部分。`.agents/skills/...` 是给支持 Agent Skills 目录结构的工具使用的兼容入口。

## 如何导入

如果你的工具支持直接导入技能包：

1. 下载或克隆这个仓库。
2. 选择整个 `premium-html-templates` 文件夹作为技能包。
3. 确认根目录能看到 `SKILL.md`。
4. 确认根目录能看到 `references/` 文件夹。

如果你的工具支持 `.agents/skills` 结构，也可以直接读取：

```text
.agents/skills/cose-html-theme-replica/SKILL.md
```

## 两种输出模式

### deck 模式

适合 PPT、slides、分页汇报、阶段复盘、提案路演、一页一屏的正式汇报。

核心约束：

- 每一页都是独立 `16:9` 画布。
- 内容必须完整放在当前页内。
- 禁止用内部滚动、缓冲区、溢出裁切来解决内容装不下的问题。
- 每页标题需要和上方保留安全间距。
- 如果内容太多，应拆页、删减、重写或降低信息密度。

### single-page 模式

适合 dashboard、单页总览、信息卡页面、资源盘点、渠道对比、长页面内容组织。

核心约束：

- 页面整体不强制 `16:9`。
- 页面和卡片高度可以随内容自然增长。
- 仍然要保持清晰的 section、网格、卡片和标题层级。
- 不要把单页模块强行塞进 PPT 卡片。

## 三种风格

### light-report

白底商务汇报风格。

适合：

- 正式汇报
- 经营复盘
- 项目提案
- SOP
- 排期规划
- 数据总结

视觉特征：

- 白色或浅灰背景
- 大标题
- 红、蓝、绿作为重点强调色
- 细边框卡片
- 数据指标、目标卡、阶段拆解

推荐组合：

```text
deck 模式 + light-report
single-page 模式 + light-report
```

### dark-ip

深色科技 IP 风格。

适合：

- 个人 IP
- 品牌策略
- 视觉体系
- 形象升级
- 科技感方案

视觉特征：

- 深蓝黑背景
- 冷色高光
- 科技感卡片
- 强对比标题
- 更适合策略感、品牌感、未来感表达

推荐组合：

```text
deck 模式 + dark-ip
```

### light-glass

浅色玻璃拟态信息卡风格。

适合：

- 资源盘点
- 渠道对比
- 商务沟通
- 报价解读
- 招商信息
- 多模块信息陈列

视觉特征：

- 浅色背景
- 半透明玻璃卡片
- 柔和蓝青渐变
- 轻量信息卡
- 适合单页和卡片化内容

推荐组合：

```text
single-page 模式 + light-glass
```

## 推荐调用方式

### 生成正式汇报 PPT 风格 HTML

```text
请使用这个 HTML 风格技能包，生成一个 deck 模式 + light-report 风格的 HTML 汇报。

主题：视频号数据复盘
要求：
1. 使用 16:9 分页 deck 结构
2. 每页内容必须完整落在画布内
3. 不要使用内部滚动、缓冲区或裁切
4. 每页标题和上方保留安全间距
5. 输出一个可直接打开的 HTML 文件
```

### 生成深色科技 IP 方案

```text
请使用这个 HTML 风格技能包，生成一个 deck 模式 + dark-ip 风格的 HTML 方案页。

主题：老板个人 IP 视觉升级方案
要求：
1. 深色科技感
2. 强标题、强对比、冷色高光
3. 每页保持 16:9
4. 内容太多时主动拆页
5. 不要添加测试页标题或风格说明外壳
```

### 生成单页信息卡页面

```text
请使用这个 HTML 风格技能包，生成一个 single-page 模式 + light-glass 风格的信息卡页面。

主题：渠道资源盘点
要求：
1. 不要强制 16:9
2. 页面和卡片高度可以随内容自然增长
3. 使用浅色玻璃拟态卡片
4. 保持清晰的 section、网格和卡片节奏
5. 输出一个可直接打开的 HTML 文件
```

## 生成时不要做什么

默认不要生成这些内容：

- 顶部测试说明栏
- 风格演示导航按钮
- “这是第几种风格”的外层说明区
- 和正式内容无关的 demo 壳
- 暴露内部来源、历史文件名或参考路径

除非明确要求做“风格总览页”或“样式展示页”，否则页面应该直接进入正式内容。

## 内容太多时怎么处理

如果是 `deck` 模式：

- 不要缩到看不清。
- 不要让内容溢出 16:9 卡片。
- 不要给单页内部加滚动条。
- 应该拆成更多页，或压缩文案，或降低模块数量。

如果是 `single-page` 模式：

- 可以让页面自然变长。
- 可以让卡片随内容增高。
- 但不要把一个卡片写成大段文章。
- 应该拆成多个 section 或多个卡片。

## 分享给别人时的最小包

如果只想分享必要文件，至少保留：

```text
SKILL.md
references/
```

如果对方工具支持 `.agents/skills`，建议完整保留整个仓库结构。

## 更新和扩展

新增风格时，推荐这样扩展：

1. 在 `references/archetypes-index.md` 里增加新风格索引。
2. 新建 `references/variant-新风格名.md`。
3. 在新 variant 文件里写清楚适用场景、配色、背景、字体、卡片、布局和禁止事项。
4. 不要把 `16:9` 或单页自适应规则写死进 variant。
5. 页面模式规则继续放在 `mode-deck.md` 和 `mode-single-page.md`。

这样后续可以持续增加新风格，同时保持主技能稳定。
