# Mode: single-page

用于单页概览、dashboard、信息卡页面、资源盘点、渠道对比、长页面内容组织。

## 核心原则

- 页面整体不强制 16:9。
- 页面高度允许随内容自然增长。
- 卡片高度可以不一致，按内容自然展开。
- 但页面仍必须有清晰 section、网格、留白和标题层级。

## 页面结构

- 首屏应有明确标题区，但不要加无关的演示说明壳。
- 主容器建议设置最大宽度，如 `1200px - 1440px`。
- 内容按 section 分组，不要无尽堆叠散卡片。
- 每个 section 应有标题、简短说明或明确的信息任务。

## 卡片规则

- 卡片允许自适应高度，不要求每张卡片等高。
- 单张卡片不应变成超长文章。
- 超长内容优先拆成多个卡片或多个 section。
- 同一行卡片允许不同高度，但整体要有明确对齐方式和视觉节奏。

推荐单卡结构：

- 1 个标题
- 1 段说明
- 1 组数据、列表、标签、表格或图示

## 字号与节奏

- 页面主标题可大，但不要覆盖首屏主体内容。
- section 标题要稳定，不要每个 section 都使用封面级大标题。
- 正文保持易读，避免大段堆叠。
- 卡片之间保持稳定间距，section 之间要有更明显的留白。

## 允许与禁止

允许：

- 页面纵向增长
- 卡片高度按内容变化
- 长表格分 section 展示
- dashboard 式多模块组合

禁止：

- 把单页模块强行塞进 16:9 卡片
- 为了对齐而裁切正文
- 为了模拟 PPT 而让每个 section 都固定一屏
- 单张卡片承载过多无分组正文

## 生成前自检

输出前进行一次结构自检：

- 页面是否有清晰 section？
- 卡片是否按信息任务分组？
- 是否存在单卡过长且无分组？
- 是否错误套用了 16:9 PPT 约束？
- 是否出现与实际内容无关的测试说明壳？

任一项为“是”，先重组内容。
