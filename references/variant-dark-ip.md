# Variant: dark-ip

## 设计目标

做出“深色、科技、克制、品牌感强”的策略视觉语言。它常用于 `deck` 模式，也可以用于深色单页品牌概览。重点不是炫技，而是高密度信息在深色体系中的高级呈现。

## 页面骨架

- 如果使用 `deck` 模式，尺寸和分页约束以 [mode-deck.md](mode-deck.md) 为准。
- 如果使用 `single-page` 模式，自然高度和卡片伸展规则以 [mode-single-page.md](mode-single-page.md) 为准。
- 页面一般由以下模块组成：
  - Cover hero
  - 目录
  - 核心洞察
  - 标签/特征卡片
  - 风格矩阵
  - 流程流转图
  - 表格或应用场景页
  - Closing

## 配色 token

- 主背景：`#0b1020`
- 背景层：`#121935`
- 文本主色：`#f5f7ff`
- 文本次色：`#b6bfd8`
- 线条：`rgba(255, 255, 255, 0.14)`
- 主强调：`#7c9cff`
- 辅助强调：`#66e0c2`
- 第三强调：`#ffd166`
- 风险色：`#ff7b7b`

## 背景规则

- 背景必须分层：
  - 深底渐变
  - 两处以上径向辉光
  - 可选细网格遮罩
- 背景氛围要明显，但主体内容对比度必须足够。

## 标题系统

- 封面标题可以更有冲击力，但必须遵守当前 mode 的字号边界
- 标题字距偏紧，行高低
- eyebrow/kicker 常用冷色强调
- 页码和导航要弱化处理，不能抢主标题

## 模块语言

- 主要靠半透明卡片和边框建立层次
- 卡片圆角偏大，通常 `20px+`
- 常见组件：
  - pill
  - metric
  - tag
  - image-box
  - flow-step
  - table-wrap
  - footer-note

## 布局习惯

- 常见 `grid-2`、`grid-3`、`grid-4`
- hero 页一般左文右视觉，或左视觉右文
- 卡片间距均匀，节奏稳定
- 装饰元素可以超出版心，但内容区仍要稳

## 适合的文案气质

- 抽象能力强
- 有“策略命名感”
- 适合品牌、形象、方向、审美判断、路径设计
- 文案可以更高级、更凝练

## 禁止事项

- 不要做成游戏 UI
- 不要使用过多高饱和荧光色
- 不要让每个卡片颜色都不同
- 不要使用太厚重的拟物阴影
