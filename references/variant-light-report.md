# Variant: light-report

## 设计目标

做出“正式、干净、锐利、适合汇报”的白底商务方案视觉语言。它既可以用于 `deck` 模式的分页 PPT，也可以用于 `single-page` 模式的商务概览或 dashboard。

## 页面骨架

- 如果使用 `deck` 模式，尺寸和分页约束以 [mode-deck.md](mode-deck.md) 为准。
- 如果使用 `single-page` 模式，自然高度和卡片伸展规则以 [mode-single-page.md](mode-single-page.md) 为准。
- 常见内容结构：
  - Cover
  - Section page
  - 2 栏内容页
  - 3 栏流程/SOP 页
  - 对比表 / 时间轴 / 卡片矩阵

## 配色 token

- 主底色：`#F8FAFC` 或更接近 `slate-50`
- 主文字：`#0f172a` 到 `#1e293b`
- 主强调红：`#E62918` 或 `#C8102E`
- 辅助蓝：`#3b82f6`
- 辅助橙：`#f97316`
- 辅助青：`#0d9488`
- 卡片边框：`rgba(226, 232, 240, 0.8)`
- 弱背景块：`slate-50`、`red-50/20`、`orange-50/40`

## 背景规则

- 整体是浅底，不是纯白死板背景。
- 使用极淡的径向弥散：
  - 左上偏红
  - 右下偏蓝
- 强度要轻，不能抢内容。

## 标题系统

- 封面主标题大、黑、紧凑、字距偏紧
- 常用 `font-black`、`tracking-tighter`
- 副标题可带一点 italic，但只用于封面或强调句
- 章节标题使用左侧竖向色条：
  - `.section-accent-title`
  - `::before` 做 5px 左色条

## 模块语言

- 区块强调靠：
  - 左边框
  - 浅色底
  - 小标签
  - 粗标题
- 常用模块：
  - 推荐结论块
  - 理由列表
  - 时间轴
  - 步骤 SOP
  - 表格对比
  - 图片占位区

## 布局习惯

- `deck` 模式中，页眉高度固定，标题和页码分离
- `single-page` 模式中，section 标题和卡片组保持清晰分区
- 主体多用 `grid grid-cols-2` 或 `grid-cols-3`
- 每页信息密度中高，但要靠大边距维持高级感
- 常用大留白：
  - 页边距 `p-10` 到 `p-12`
  - 模块间距 `gap-10` 到 `gap-16`

## 适合的文案气质

- 结论先行
- 结构明确
- 有命名感，如“核心判断”“执行表”“矩阵”“SOP”
- 不要太营销化，不要口语化过头

## 禁止事项

- 不要大面积渐变按钮风
- 不要赛博霓虹
- 不要使用圆润可爱组件语言
- 不要把页面做成普通博客排版
