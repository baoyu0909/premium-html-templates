# Prompt Recipes

下面这些模板用于在调用 skill 时，稳定指定风格、结构和输出目标。

## 1. 白底商务汇报

```text
请基于 cose-html-theme-replica 生成一个 deck 模式 + light-report 风格的 HTML 汇报页。
主题是：{{主题}}
目标受众是：{{受众}}
内容模块包括：{{模块列表}}

要求：
1. 风格贴近内部既有白底商务汇报体系
2. 使用 16:9 滚动式 PPT 结构
3. 每页内容必须完整落入 16:9 卡片内，不能靠内部滚动、缓冲区或裁切兜底
4. 每页标题区必须保留顶部安全区和标题下方呼吸空间
5. 遵守 deck 模式字号边界，标题过长时先改写或拆页
6. 保持白底商务、高级、强章节感
7. 重点还原标题系统、左侧章节色条、卡片密度和留白节奏
8. 不要在产物中暴露内部参考来源
9. 不要额外添加测试说明栏、总导航壳或风格介绍外框
10. 直接输出可运行 HTML
```

## 2. 深色科技 IP

```text
请基于 cose-html-theme-replica 生成一个 deck 模式 + dark-ip 风格的 HTML deck。
主题是：{{主题}}
希望表达的气质是：{{气质关键词}}
内容模块包括：{{模块列表}}

要求：
1. 风格贴近内部既有深色科技 IP 体系
2. 使用深色科技感、半透明卡片、冷色辉光背景
3. 每页内容必须完整落入 16:9 卡片内，不能靠内部滚动、缓冲区或裁切兜底
4. 每页标题区必须保留顶部安全区和标题下方呼吸空间
5. 标题要有品牌感和策略感，但必须遵守 deck 模式字号边界
6. 页面结构优先使用封面、矩阵、卡片组、流程图、表格这几类模块
7. 不要在产物中暴露内部参考来源
8. 不要额外添加测试说明栏、总导航壳或风格介绍外框
9. 直接输出可运行 HTML
```

## 3. 浅色玻璃拟态信息卡

```text
请基于 cose-html-theme-replica 生成一个 single-page 模式 + light-glass 风格的 HTML 页面。
主题是：{{主题}}
需要对比的对象是：{{对象}}
核心信息包括：{{信息列表}}

要求：
1. 风格贴近内部既有浅色玻璃拟态信息卡体系
2. 使用浅色弥散背景和玻璃卡片
3. 页面高度和卡片高度允许随内容自然增长，不要强套 16:9
4. 页面以双栏或多卡片信息组织为主
5. 每个 section 和卡片必须有清晰信息任务，超长内容拆卡片
6. 适合商务沟通、资源汇总或报价解读
7. 不要在产物中暴露内部参考来源
8. 不要额外添加测试说明栏、总导航壳或风格介绍外框
9. 直接输出可运行 HTML
```

## 4. 单页商务数据简报

```text
请基于 cose-html-theme-replica 生成一个 single-page 模式 + light-report 风格的 HTML 数据简报。
主题是：{{主题}}
核心指标包括：{{指标列表}}
内容模块包括：{{模块列表}}

要求：
1. 保留白底商务汇报的配色、标题和卡片语言
2. 页面和卡片高度随内容自然增长，不强制 16:9
3. 用 section 组织内容，每个 section 有明确标题和信息任务
4. 超长内容拆成多个卡片或 section
5. 不要额外添加测试说明栏、总导航壳或风格介绍外框
6. 直接输出可运行 HTML
```

## 5. 新增变体时的提示词模板

```text
请基于现有 cose-html-theme-replica skill 扩展一个新的风格变体。
新变体名称：{{variant-name}}
参考来源：{{来源文件或视觉描述}}
适用场景：{{场景}}
目标气质：{{气质}}

请完成：
1. 在 archetypes-index.md 中注册新变体
2. 新建对应的 variant-*.md reference
3. 定义色彩 token、背景规则、标题规则、卡片规则、布局骨架、禁止事项
4. 不要把 16:9 或单页自适应规则写死进 variant；尺寸规则应放进 mode 文件
5. 保持 SKILL.md 主流程不膨胀
```
